## 1. Εισαγωγή & Φιλοσοφία Συστήματος

Το **υποσύστημα επισκευών** του web-Praxis είναι το πλέον χρησιμοποιούμενο μέρος ενός ΤΒΙΤ. Παρακολουθεί **ολόκληρη** την πορεία μιας βλάβης: από την πρώτη αναγγελία μέχρι την τελική καταγραφή αποτελεσμάτων.

### Βασικές Αρχές

```
┌─────────────────────────────────────────────────────────────┐
│  ΑΡΧΗ 1: Κάθε βλάβη = κωδικός + ιστορικό + κατάσταση      │
│  ΑΡΧΗ 2: Δεν χρειάζονται ΟΛΑ τα βήματα για κάθε βλάβη    │
│  ΑΡΧΗ 3: Το σύστημα ενημερώνει αυτόματα τους εμπλεκόμενους│
│  ΑΡΧΗ 4: Αυτόματη μεταφορά στοιχείων μεταξύ βημάτων       │
└─────────────────────────────────────────────────────────────┘
```

### Ποιος κάνει τι;

| Ρόλος | Αρμοδιότητα |
|-------|-------------|
| **Προσωπικό τμήματος/κλινικής** | Αναγγελία βλάβης (Βήμα 1) |
| **Υπεύθυνος ΤΒΙΤ** | Διαχείριση, ανάθεση, εγκρίσεις (Βήματα 2–6) |
| **Τεχνικός ΒΙΤ** | Εκτέλεση επισκευής, καταγραφή αποτελεσμάτων |

> ⚠️ **Σημαντικό:** Η αναγγελία βλάβης **ΔΕΝ** σχετίζεται με χρέωση μηχανημάτων. Είναι εργαλείο **ταχύτερης αποκατάστασης**.

## 3. Καταστάσεις Βλαβών — Χρωματικός Κώδικας

Κάθε βλάβη έχει **χρώμα** στη λίστα για άμεση εποπτεία:

```
🟢 ΠΡΑΣΙΝΟ   = Νέες Αναγγελίες
               → Αναγγέλθηκαν, καμία ενέργεια από ΤΒΙΤ ακόμα

🟠 ΠΟΡΤΟΚΑΛΙ = Σε Εκκρεμότητα
               → ΤΒΙΤ έχει κάνει: Κλήση Εταιρείας ή Ανάθεση ή Μετακίνηση (Βήματα 3,7)

🔵 ΜΠΛΕ      = Αναμονή Έγκρισης
               → Έχει υποβληθεί Αίτημα Χορήγησης Έγκρισης (Βήμα 5)

🩷 ΡΟΖ       = Αναμονή Επισκευής
               → Έχει ληφθεί Έγκριση (ΑΔΑ) — περιμένει εκτέλεση

⬜ ΓΚΡΙ      = Ολοκληρωμένες
               → Έχουν ολοκληρωθεί: αποκατάσταση + αρχεία + κόστη
```

> 💡 **Pro tip:** Χρησιμοποίησε τα φίλτρα κατάστασης στην κορυφή της λίστας για γρήγορη εποπτεία. Το πορτοκαλί σημαίνει "κάτι τρέχει" — το πράσινο σημαίνει "κανείς δεν το έχει δει ακόμα".

## 4. Βήμα 1 — Αναγγελία Βλάβης

### Πώς να μπεις στη φόρμα (2 τρόποι)

```
ΤΡΟΠΟΣ Α: Πλάγιο Μενού
  Επισκευές → Αναφορά Βλάβης

ΤΡΟΠΟΣ Β: Από καρτέλα μηχανήματος
  Μητρώο → [επιλογή μηχανήματος] → κουμπί "Αναφορά βλάβης"
  ✨ Πλεονέκτημα: Τα στοιχεία μηχανήματος συμπληρώνονται ΑΥΤΟΜΑΤΑ
```

### Πεδία Φόρμας

| Πεδίο | Συμπλήρωση | Σημείωση |
|-------|-----------|---------|
| Μονάδα Υγείας χρήσης | Αυτόματο / Χειροκίνητο | * Υποχρεωτικό |
| **Μηχάνημα** | Αναζήτηση (κωδικός/μοντέλο/κατασκευαστής/σειριακό) | * Υποχρεωτικό |
| Προτεραιότητα | χαμηλή / κανονική / υψηλή | Default: κανονική |
| Τμήμα εντοπισμού | Αναζήτηση | * Υποχρεωτικό |
| **Περιγραφή** | Ελεύθερο κείμενο | * Υποχρεωτικό — το βλέπει το ΤΒΙΤ |
| Κατάσταση Λειτουργίας | Εκτός Χρήσης / Σε Χρήση | Default: Εκτός Χρήσης |
| Ημ/νία Εντοπισμού | Ημερολόγιο | Προαιρετικό — διαφορετικό από καταχώριση |
| Ημ/νία Καταχώρισης | **Αυτόματο** | ΔΕΝ αλλάζει |
| Όνομα Αναγγέλλοντος | **Αυτόματο** | Από το login |

### Αναζήτηση Μηχανήματος

Μπορείς να ψάξεις με οποιοδήποτε από τα εξής:
```
- Κωδικό INBIT
- Κατασκευαστή (π.χ. "SIEMENS")
- Μοντέλο (π.χ. "BORA UP 2080")
- Σειριακό Αριθμό
- Ομάδα μηχανήματος
- Τμήμα / Κλινική
```

### Μετά την Αποθήκευση

```diff
+ Η βλάβη παίρνει αυτόματα ΚΩΔΙΚΟ ΒΛΑΒΗΣ (π.χ. 113)
+ Ο κωδικός τη συνοδεύει σε ΟΛΗ την πορεία της
+ Αυτόματη ειδοποίηση στο ΤΒΙΤ (εικονίδιο 🔔 στο μενού)
+ Η βλάβη εμφανίζεται με ΠΡΑΣΙΝΟ χρώμα στη λίστα
```

### Τρόποι Αναγγελίας

```
1. Μέσω web-Praxis (browser/desktop)
2. Μέσω mobile app (ειδική εφαρμογή κινητού)

   ⚠️ Και στους δύο τρόπους: ο χρήστης πρέπει να έχει
   δηλωθεί ΩΣ ΧΡΗΣΤΗΣ ΑΝΑΓΓΕΛΙΩΝ στο σύστημα.
```

## 5. Βήμα 2 — Διαχείριση Βλαβών

### Πώς να μπεις (2 τρόποι)

```
ΤΡΟΠΟΣ Α: Πλάγιο Μενού
  Επισκευές → Διαχείριση Βλαβών

ΤΡΟΠΟΣ Β: Dashboard (αρχική σελίδα)
  Κουτί "Αναγγελίες Βλαβών" (με το ⚠️ και τον αριθμό)
```

### Τι βλέπεις στη λίστα

Κάθε γραμμή περιέχει:
- Κατάσταση Βλάβης (χρωματιστό)
- Μονάδα Υγείας
- Προτεραιότητα
- Κωδικός Βλάβης
- Μηχάνημα
- Ημ/νία Καταχώρισης
- Τμήμα εντοπισμού
- Υπεύθυνος βλάβης
- Περιγραφή

### Φίλτρα Αναζήτησης

```
Απλή Αναζήτηση:
  [Όλες] [Νέες Αναγγελίες] [Σε Εκκρεμότητα] [Αναμονή Έγκρισης]
  [Αναμονή Επισκευής]      [Ολοκληρωμένες]

Σύνθετη Αναζήτηση:
  - Μονάδα υγείας χρήσης / χρέωσης / κτήσης
  - Κωδικός Βλάβης
  - Μηχάνημα
```

### Καρτέλα Συγκεκριμένης Βλάβης

Κάνε κλικ σε μια εγγραφή → ανοίγει η καρτέλα της βλάβης.

**Επεξεργάσιμα πεδία:**

| Πεδίο | Τι είναι |
|-------|---------|
| **Σχόλια βλάβης ΤΒΙΤ** | Εσωτερικές σημειώσεις του ΤΒΙΤ |
| **Υπεύθυνος βλάβης** | Ο υπάλληλος ΤΒΙΤ που παρακολουθεί (≠ τεχνικός) |

> ⚠️ **Σύγχυση που πρέπει να αποφύγεις:**
> - **Υπεύθυνος βλάβης** = παρακολουθεί την πορεία (διοικητικός ρόλος)
> - **Τεχνικός ΒΙΤ** = εκτελεί την επισκευή (τεχνικός ρόλος)
> Είναι δύο διαφορετικά πρόσωπα!

### Κουμπιά Ενεργειών (κάτω της καρτέλας)

```
[ Κλήση ] [ Ανάθεση ] [ Μετακίνηση ] [ Αίτημα χορήγησης έγκρισης ]
[ Έγκριση ] [ Άλλη Ενέργεια ] [ Καταγραφή Αποτελεσμάτων ]
[ Εκτύπωση Δελτίου Βλάβης 🖨️ ]
```

## 6. Βήμα 3α — Ανάθεση σε ΤΒΙΤ

Χρησιμοποιείται όταν η επισκευή γίνεται **εσωτερικά** από προσωπικό ΒΙΤ.

### Διαδικασία

```
1. Άνοιξε την καρτέλα βλάβης από Διαχείριση Βλαβών
2. Πάτα κουμπί [Ανάθεση]
3. Στο πεδίο "Ανατέθηκε σε:" → 📁 εύρεση → επίλεξε τεχνικό(ύς)
4. Ημ/νία Ανάθεσης: συμπληρώνεται ΑΥΤΟΜΑΤΑ
5. Πρόσθεσε σχόλια αν χρειάζεται
6. [Αποθήκευση]
```

### Αποτέλεσμα

```diff
+ Αυτόματη ειδοποίηση στον τεχνικό ΒΙΤ (εικονίδιο 🔧 στο μενού)
+ Ο τεχνικός βλέπει τις λεπτομέρειες πατώντας το εικονίδιο
+ Βλάβη → κατάσταση "Σε εκκρεμότητα" (πορτοκαλί)
```

### Σημαντικές Επισημάνσεις

```
✅ Ο τεχνικός ΔΕΝ χρειάζεται login αν δεν έχει πρόσβαση —
   αρκεί να είναι καταχωρισμένος στο σύστημα

✅ Η ανάθεση μπορεί να γίνει ΠΟΛΛΕΣ ΦΟΡΕΣ
   (π.χ. πρώτα ΒΙΤ, μετά αδυναμία → κλήση εταιρείας)

⚠️ Αν ο τεχνικός δεν φαίνεται στη λίστα → επικοινωνία με INBIT
```

## 7. Βήμα 3β — Κλήση Εταιρείας / Έρευνα Αγοράς

Χρησιμοποιείται όταν:
- Δεν υπάρχει σύμβαση → χρειάζεται έρευνα αγοράς
- Αδυναμία εσωτερικής επισκευής → εξωτερική ανάθεση

### Διαδικασία

```
1. Άνοιξε την καρτέλα βλάβης
2. Πάτα κουμπί [Κλήση]
3. Πεδίο "Εταιρεία": αναζήτηση με επωνυμία ή ΑΦΜ
4. Ημ/νία Κλήσης: αυτόματο
5. Σχόλια: προαιρετικά
6. [Αποθήκευση]
7. Εκτύπωση εγγράφου αναγγελίας/έρευνας αγοράς (προσυμπληρωμένο)
```

### ⚠️ ΚΡΙΣΙΜΗ ΕΠΙΣΗΜΑΝΣΗ

```
╔══════════════════════════════════════════════════════════╗
║  Η "Κλήση" ΔΕΝ ενημερώνει αυτόματα την εταιρεία!      ║
║  Πρέπει να επικοινωνήσεις μαζί της ΕΣΩΤΕΡΙΚΑ.          ║
╚══════════════════════════════════════════════════════════╝
```

### Σύγκριση Κλήση vs. Ανάθεση

| | Κλήση Εταιρείας | Ανάθεση ΒΙΤ |
|---|---|---|
| Ενημερώνει αυτόματα; | ❌ ΟΧΙ | ✅ ΝΑΙ (ειδοποίηση) |
| Σε ποιον; | Εξωτερική εταιρεία | Τεχνικός ΒΙΤ |
| Έγγραφο; | Έγγραφο έρευνας αγοράς | — |
| Κατάσταση βλάβης; | Σε εκκρεμότητα 🟠 | Σε εκκρεμότητα 🟠 |

### Έγγραφο Έρευνας Αγοράς

Πατώντας εκτύπωση, λαμβάνεις έγγραφο **προσυμπληρωμένο** με:
- Στοιχεία νοσοκομείου
- Στοιχεία μηχανήματος
- Περιγραφή βλάβης

> Αποστέλλεται στην εταιρεία για να δώσει προσφορά.

## 8. Βήματα 5 & 6 — Αίτημα Έγκρισης & Έγκριση

Απαιτούνται **μόνο** όταν υπάρχει επιπλέον κόστος (π.χ. ανταλλακτικά, εξωτερική επισκευή χωρίς σύμβαση).

### Πότε ΔΕΝ χρειάζονται

```
✅ Σύμβαση συντήρησης που καλύπτει τα πάντα → Παράλειψε 5 & 6
✅ Εγγύηση → Παράλειψε 5 & 6
✅ Εσωτερική επισκευή χωρίς αγορά ανταλλακτικών → Παράλειψε 5 & 6
```

### Πότε χρειάζονται

```
⚠️ Εξωτερική εταιρεία χωρίς σύμβαση → 5 & 6 ΥΠΟΧΡΕΩΤΙΚΑ
⚠️ Σύμβαση αλλά extra κόστος (ανταλλακτικά/εργασία) → 5 & 6
```

### Βήμα 5 — Αίτημα Χορήγησης Έγκρισης

```
1. Από καρτέλα βλάβης → [Αίτημα χορήγησης έγκρισης]
2. Συμπλήρωσε στοιχεία κόστους
3. Εκτύπωση "Εγγράφου Αίτησης Έγκρισης Επισκευής"
4. Αποθήκευση → Βλάβη: "Αναμονή Έγκρισης" 🔵

→ Το έγγραφο πάει για υπογραφή (ΑΔΑ Εργασίας/Ανταλλακτικών)
```

### Βήμα 6 — Έγκριση

```
1. Μετά τη λήψη ΑΔΑ → [Έγκριση]
2. Καταχώριση ΑΔΑ
3. Αποθήκευση → Βλάβη: "Αναμονή Επισκευής" 🩷
```

## 9. Βήματα 7 & 8 — Αποστολή & Επιστροφή Μηχανήματος

Χρησιμοποιούνται όταν το μηχάνημα **φεύγει από το νοσοκομείο** για επισκευή.

```
Βήμα 7 — Αποστολή:
  → [Μετακίνηση] → τύπος: Αποστολή
  → Εκτύπωση εγγράφου αποστολής (προσυμπληρωμένο)
  → Βλάβη: "Σε εκκρεμότητα" 🟠

Βήμα 8 — Επιστροφή:
  → [Μετακίνηση] → τύπος: Επιστροφή
  → Καταγραφή ημερομηνίας επιστροφής
```

## 10. Βήμα 9 — Καταγραφή Αποτελεσμάτων

### Το πιο σημαντικό βήμα — ΠΑΝΤΑ υποχρεωτικό

```
1. Από καρτέλα βλάβης → [Καταγραφή Αποτελεσμάτων]
2. Συμπλήρωσε:
   - Αποτέλεσμα επισκευής
   - Κόστος (εργασία + ανταλλακτικά)
   - Ημερομηνία ολοκλήρωσης
3. Επισύναψε αρχεία: τιμολόγια, δελτία επισκευής, κτλ.
4. Αποθήκευση → Βλάβη: "Ολοκληρωμένη" ⬜
```

### Μαζικό Ανέβασμα Αρχείων

Αν έχεις **πολλές βλάβες** με ίδιο τύπο αρχείου:

```
1. Στη λίστα βλαβών: επίλεξε τις εγγραφές (checkboxes)
2. Κλικ στο εικονίδιο ☁️ "Μαζικό ανέβασμα αρχείων"
3. Σύρε τα αρχεία στο παράθυρο διαλόγου
→ Τα αρχεία εφαρμόζονται σε ΟΛΑ τα επιλεγμένα
```

## 11. Πρακτικά Σενάρια

### 🅐 Σενάριο Α — Απλή Εσωτερική Επισκευή

> *Βλάβη σε monitor, επισκευάζεται από τεχνικό ΒΙΤ χωρίς ανταλλακτικά.*

```
1 → Αναγγελία Βλάβης
2 → Διαχείριση (ορισμός υπεύθυνου)
3α → Ανάθεση σε τεχνικό ΒΙΤ
9 → Καταγραφή Αποτελεσμάτων + επισύναψη δελτίου επισκευής

Σύνολο βημάτων: 4
```

### 🅑 Σενάριο Β — Εξωτερική Επισκευή με Κόστος

> *Βλάβη σε αναλυτή, δεν υπάρχει σύμβαση, απαιτείται εξωτερική επισκευή.*

```
1 → Αναγγελία Βλάβης
2 → Διαχείριση (ορισμός υπεύθυνου)
3β → Κλήση εταιρείας + εκτύπωση εγγράφου έρευνας αγοράς
4 → Έρευνα αγοράς (συλλογή προσφορών, scan)
5 → Αίτημα Χορήγησης Έγκρισης + εκτύπωση + υπογραφές (ΑΔΑ)
6 → Έγκριση (καταχώριση ΑΔΑ)
7 → Αποστολή μηχανήματος + εκτύπωση εγγράφου αποστολής
8 → Επιστροφή μηχανήματος
9 → Καταγραφή Αποτελεσμάτων + τιμολόγια + δελτία

Σύνολο βημάτων: 9
```

### 🅒 Σενάριο Γ — Σύμβαση αλλά Ανταλλακτικά

> *Υπάρχει σύμβαση αλλά χρειάζονται ανταλλακτικά εκτός σύμβασης.*

```
1 → Αναγγελία Βλάβης
2 → Διαχείριση
3β → Κλήση εταιρείας (εξουσιοδοτημένος προτιμάται!)
5 → Αίτημα Έγκρισης (μόνο για το extra κόστος ανταλλακτικών)
6 → Έγκριση
9 → Καταγραφή Αποτελεσμάτων

Σύνολο βημάτων: 6
```

## 14. Κοινά Λάθη & Παγίδες

### ❌ Λάθος 1: Ξεχνώ να κάνω Καταγραφή Αποτελεσμάτων

```diff
- Η βλάβη μένει "ανοιχτή" για πάντα
- Στατιστικά νοσοκομείου αλλοιώνονται
- Ιστορικό μηχανήματος ελλιπές

+ Η Καταγραφή Αποτελεσμάτων είναι ΠΑΝΤΑ το τελευταίο βήμα
```

### ❌ Λάθος 2: Νομίζω ότι η "Κλήση" ειδοποιεί την εταιρεία

```diff
- Η εταιρεία ΔΕΝ λαμβάνει αυτόματη ειδοποίηση
- Η κλήση είναι απλώς ΚΑΤΑΓΡΑΦΗ της επαφής

+ Επικοινώνησε με την εταιρεία ΕΣΩΤΕΡΙΚΑ
+ Στείλε το έγγραφο έρευνας αγοράς χειροκίνητα
```

### ❌ Λάθος 3: Συγχέω "Υπεύθυνο βλάβης" με "Τεχνικό ΒΙΤ"

```diff
- Υπεύθυνος βλάβης = διοικητικός, παρακολουθεί
- Τεχνικός ΒΙΤ = εκτελεί την επισκευή

+ Είναι δύο ξεχωριστά πεδία στο σύστημα
```

### ❌ Λάθος 4: Κάνω Αίτημα Έγκρισης ενώ δεν χρειάζεται

```diff
- Αν υπάρχει σύμβαση χωρίς extra κόστος → ΔΕΝ χρειάζεται
- Αν υπάρχει εγγύηση → ΔΕΝ χρειάζεται

+ Έλεγξε αν υπάρχει σύμβαση ΠΡΩΤΑ
```

### ❌ Λάθος 5: Ψάχνω τεχνικό που δεν υπάρχει στη λίστα

```diff
- Ο τεχνικός ΔΕΝ εμφανίζεται στην Ανάθεση

+ Επικοινώνησε με την ομάδα INBIT
+ Ο τεχνικός πρέπει να καταχωριστεί πρώτα στο σύστημα
```
